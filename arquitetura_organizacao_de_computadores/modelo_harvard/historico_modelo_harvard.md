## 🏛 Histórico da Arquitetura Harvard

A **Arquitetura Harvard** é um modelo de organização de computadores que se caracteriza pela **separação física entre memória de dados e memória de instruções**. Diferente da arquitetura de Von Neumann, ela utiliza caminhos (barramentos) distintos para cada tipo de informação.

---

## 📜 Origem do Nome

O nome vem do computador:

* Harvard Mark I

Desenvolvido em 1944 na Universidade de Harvard em parceria com a IBM.

---

## 🧠 Desenvolvimento Histórico

### 🔹 1944 – Harvard Mark I

* Criado sob liderança de Howard Aiken
* Utilizava separação entre instruções e dados
* Era eletromecânico (não totalmente eletrônico)

Esse modelo inspirou o conceito da Arquitetura Harvard.

---

### 🔹 Diferença em relação ao modelo de John von Neumann

Enquanto Von Neumann propôs que dados e instruções compartilhassem a mesma memória, a Arquitetura Harvard defendia:

* Memória separada
* Barramentos separados
* Possibilidade de acessar dados e instruções simultaneamente

---

## ⚙ Características Técnicas

✔ Memória de dados separada da memória de instruções
✔ Barramento de dados separado do barramento de instruções
✔ Maior desempenho
✔ Maior segurança contra certos tipos de ataques

---

## 🚀 Evolução Moderna

Hoje, a maioria dos processadores utiliza um modelo híbrido chamado:

### 🧩 Arquitetura Harvard Modificada

Ela mantém:

* Memórias cache separadas para dados e instruções (L1)
* Memória principal unificada

Presente em:

* Microcontroladores
* Processadores modernos
* Sistemas embarcados

Exemplo de uso em microcontroladores da:

* Microchip Technology

---

## 🔐 Segurança

Como dados e instruções ficam separados:

* É mais difícil executar código malicioso inserido como dado
* Reduz risco de ataques de injeção de código

---

# 📊 Comparação Rápida

| Característica | Harvard  | Von Neumann           |
| -------------- | -------- | --------------------- |
| Memória        | Separada | Compartilhada         |
| Barramento     | Separado | Único                 |
| Desempenho     | Maior    | Limitado pelo gargalo |
| Segurança      | Maior    | Mais vulnerável       |

---

## 🎯 Importância Atual

A Arquitetura Harvard é muito usada em:

* Sistemas embarcados
* Indústria 4.0
* IoT
* Microcontroladores
* Dispositivos médicos
